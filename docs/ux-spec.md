# UX仕様

## 基本方針

UIはスマホ中心で、ChatGPTのような会話体験をベースにする。PC表示は補助であり、スマホで自然に作品と話数を切り替えながら執筆できることを優先する。

## 画面構成

### ヘッダー

- サービス名
- 現在の作品名
- 現在の話タイトル
- ボード、本文、設定の切り替えボタン

### 作品・話数切り替えバー

- 作品セレクト
- 話数セレクト
- 新規作品
- 作品削除
- 新規話
- 話削除

削除時はブラウザの `confirm` で確認する。

制約:

- 最後の1作品は削除できない
- 各作品の最後の1話は削除できない

### フェーズ切り替え

4つのフェーズをタブとして表示する。

1. 構造化
2. 試し書き
3. 微調整
4. 本稿

フェーズを切り替えると、入力欄のプレースホルダーと入力例が切り替わる。

### チャット

中央の主領域はチャットログである。

- ユーザー発言
- AI応答
- 発言時のフェーズラベル
- 本稿フェーズのAI応答は小説本文として読みやすいスタイルにする

### 入力欄

下部固定のコンポーザー。

- フェーズごとの入力例
- テキスト入力
- 送信
- 次フェーズへ進む
- 文字数表示
- 本文パネルを開く

APIキーは入力欄には置かない。設定パネルで変更する。

## 創作ボード

創作ボードは「現在採用している作品設定」の台帳である。

項目:

- 作品タイトル
- コンセプト
- キャラクター
- 世界観・ルール
- プロット
- 参照リンク・素材メモ
- 文体・本稿ルール

AIとの会話後、ボードは自動更新される。ユーザーはいつでも手動修正できる。

フェーズ別メモは持たない。会話の途中経過はチャットログに残し、ボードには採用済みの設定だけを置く。

## 本文パネル

選択中の話の本文だけを表示する。

本文は本稿フェーズで生成されたAI応答だけを蓄積する。構造化、試し書き、微調整フェーズの応答は本文には入れない。

## 設定パネル

設定パネルはAI実行環境とAI作家ペルソナだけを扱う。

項目:

- OpenRouter APIキー
- モデル
- AI作家名
- AI作家ペルソナ・文体
- 視点
- 得意ジャンル

キャラクター、世界観、プロットは設定ではなく創作ボードで扱う。

## 色と見た目

カラーバランスはカクヨムを参考に、白と薄いグレーを基調にする。アクセントは青系に寄せる。

主要色:

- 背景: 薄いグレー
- 面: 白
- 文字: 黒に近いグレー
- 境界線: 薄いグレー
- アクセント: 青
